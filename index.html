🃏 Deck: 54 remaining

A standard 54-card deck (52 + 2 Jokers) tracks your draws across the session. Each card gives a rank (for table lookup) and a suit (for domain context). When you draw a Joker, the entire discard pile reshuffles back in and a Random Event is triggered automatically.

Suit Domains: ♣ Clubs = Physical · ♦ Diamonds = Technical · ♠ Spades = Mystical · ♥ Hearts = Social

Optional — Use Only Dice: When you would draw a card, roll a d12 for rank and a d4 for suit. On 12, flip a coin for Q or K.

🎲 Dice Roller

Roll any dice your game system calls for — skill checks, attacks, damage, saving throws, etc. Results are inserted at the cursor as a formatted blockquote. You can also use the quick buttons to roll a single die of that type.

Optional — Use Only Cards: When you'd roll a d6, draw a card instead and use rank ÷ 2 (round down). Discard Aces.

❓ Oracle (Yes/No)

Ask a simple question and choose how likely a "Yes" is. Roll 2d6 — two dice with different jobs:

  • Answer die: Compare against the threshold (Likely 3+, Even 4+, Unlikely 5+)
  • Modifier die: 1 = "but…" · 2–5 = no modifier · 6 = "and…"

Advantage (homebrew): Roll 2d6 for the answer die and take the highest. This makes "Yes" more likely without changing the modifier. Toggle it below or set as default in Settings.

Always interpret in context. Give the result meaning.

📊 Oracle (How)

Use this when you need to know how much of something there is — how big, how good, how strong, how numerous, etc. Roll a d6 and interpret the result in context.

Example: "How many guards are there?" → roll → "More than expected" → maybe 8 instead of the usual 4.

🎬 Set the Scene

Use this at the start of every new scene. First, describe where your character is and what they're trying to accomplish. Then roll a Scene Complication.

On a 5+, it's an Altered Scene — roll again on the Altered Scene table and combine both results. The scene is changed in some unexpected way.

You don't have to use the complication literally — interpret it in context. "Things actually go as planned" on a 6 means the scene plays out as expected, which can still be interesting.

🎭 GM Moves

GM Moves advance the action — they're what a GM would do to keep the game moving. Use them to describe the result as the GM normally would.

Pacing Move: Use when there's a lull in the action, or you're thinking "what happens next?" These fill in the GM's narrative role — foreshadowing, reveals, NPC actions.

Failure Move: Use when a PC fails a check and there should be consequences. These keep failures interesting rather than just "nothing happens." Not every failure needs a GM Move — only when consequences matter.

If a Pacing Move result is "Add a Random Event," a Random Event is triggered automatically.

⚡ Random Event

A Random Event injects something unexpected into the story. It draws two focus cards:

  • What happens: Action Focus
  • Involving: Topic Focus

Combine both results and interpret them together. The suit domain of each card adds colour. Random Events are also triggered automatically when you draw a Joker, or when certain GM Moves or Hex Events call for one.

🔍 Focus Tables

Draw a card and look up the rank on the appropriate table. Combine with the suit domain for a richer interpretation.

  • Action Focus — "What does it do?" (verbs: Seek, Harm, Reveal…)
  • Detail Focus — "What kind of thing is it?" (adjectives: Old, Exotic, Simple…)
  • Topic Focus — "What is this about?" (subjects: Allies, History, Enemies…)

Complex Questions: When a yes/no won't cut it, draw from the most relevant Focus table. If the answer is still insufficient, draw from a second Focus table and combine both results.

The suit domain doesn't change the word — it adds a lens. "Harm ♥ (Social)" might mean a social wound: betrayal, humiliation, or lost reputation.

⚙️ Generators

Content-neutral generators that work in any setting. Interpret results loosely and in context — the answers are prompts, not prescriptions.

  • Generic: Towns, factions, items, monsters, spaceships — anything. Draws Action + Detail + How significant.
  • Plot Hook: Objective + Adversary + Reward — a mission or quest seed.
  • NPC: Identity + Goal + Notable Feature + Attitude + Conversation topic.
  • Dungeon: First click sets the Theme (Detail + Action). Use "New Dungeon" to reset. "New Area" generates Location + Encounter + Object + Exits. First area always has 3 exits.
  • Hex: Define your region's terrain types first, then generate each new hex you explore. On an Event (5–6), a Random Event fires and you Set the Scene.

📖 Rules Reference
How to Play
  1. Create one or more characters using your chosen game system.
  2. Roll a starting Plot Hook and a Random Event, then Set the Scene.
  3. Start asking the Oracle questions. Interpret the answers in context.
  4. Play the game to overcome the challenges of the scene.
  5. Use GM Moves to move the action.
  6. Set the Scene for the next thing you want your character to do.
Tips for Best Results
  • Ask mostly yes/no questions
  • Loose interpretations are okay
  • Always go with what's cool
  • If it doesn't make sense, try again
  • Use GM Moves to drive the action
Give all results meaning. Embrace the unexpected. Reject the nonsensical.
